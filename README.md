2. `index.html` 파일을 생성하고 기본 구조를 만든 후 개발을 시작합니다.
3. 안티그래비티 에이전트에게 구현하고 싶은 기능을 설명할 때, `rules.md`에 선언된 개발 표준을 준수하며 작성하도록 상기시킵니다.

## 📝 서비스 기획서 (시네맵 - CineMap)

### 1. 유저 (Target User)
* **영화/드라마 덕후 및 성지순례 여행자(Set-jetters)**: 감명 깊게 본 작품의 배경을 직접 방문하여 주인공과 동일한 구도로 인증샷을 남기고 싶어 하는 유저.
* **감성 테마 여행 계획자**: 평범하고 식상한 관광지 투어에서 벗어나, 스토리텔링과 낭만이 결합한 특별한 여행 코스를 설계하고자 하는 유저.

### 2. 문제점 (Pain Point)
* **정보의 파편화**: 특정 촬영지의 구체적인 위치와 가는 방법, 명장면 정보를 확인하려면 블로그, 유튜브, 구글 맵을 개별적으로 검색해야 함.
* **매칭의 어려움**: 단순 텍스트 주소 정보만으로는 작품 속 어떤 신(Scene)이 실제 어느 장소에서 촬영되었는지 직관적으로 시각화되지 않음.

### 3. 해결 방안 (Solution)
- **매체**: 웹서비스 (순수 HTML/CSS/JS 바닐라 스택)
- **주요 기능 (1)**: 인터랙티브 시네마틱 SVG 월드맵
  - *설명 및 UI 구상:* 다크 모드 톤의 인터랙티브 SVG 세계 지도를 중앙에 배치합니다. 영화/드라마 촬영지 스팟에 네온 빛의 마커(Glow Marker)가 깜빡이며, 마커 클릭 시 해당 위치로 지도가 부드럽게 줌인(CSS Transform/Scale)되면서 촬영지 정보 카드가 우측 또는 하단에서 슬라이드인(Slide-in) 형태로 등장합니다.
- **주요 기능 (2)**: 작품별 필터링 및 명장면 매칭 모달
  - *설명 및 UI 구상:* 화면 상단에 작품별(예: 라라랜드, 도깨비 등), 장르별 태그 필터 버튼을 배치합니다. 특정 작품 필터 선택 시 지도 위 관련 마커만 밝게 빛나며, 상세 정보 모달 내에서 실제 촬영 현장 이미지와 작품 속 원본 캡처본을 좌우 드래그 슬라이더(Split Compare UI)로 직관적으로 교차 비교할 수 있습니다.
- **주요 기능 (3)**: '나만의 성지순례 버킷리스트' 루트 빌더
  - *설명 및 UI 구상:* 마음에 드는 촬영지 정보 카드에서 북마크 버튼을 클릭하면 로컬스토리지를 통해 나만의 가상 여행 루트에 추가됩니다. 추가된 순서에 따라 지도 위에 점선(SVG Path dash-array 애니메이션)으로 동선 라인을 그려주고 예상 경로를 함께 요약 제공합니다.

### 4. 기대효과 (Expected Effect)
* 흩어져 있던 영화/드라마 촬영지 정보를 지도 공간 위에 집약하여 여행 준비 단계에서의 정보 탐색 시간을 80% 이상 단축시킵니다.
* 영화 속 감성 스토리텔링(명대사, 명장면)을 지도와 결합하여 여행 준비 과정 자체의 만족도를 높이고, 나만의 특별한 여행 경로를 소장할 수 있습니다.
